# Introduction & Goals

TEMPO is a desktop application developed at the Deggendorf Institute of Technology (DIT) to support researchers and engineers working with the Calliope energy system modelling framework. The application removes the need to manually write YAML configuration files by providing a graphical interface for composing, managing, and executing Calliope optimization models.

The primary goals of the system are:

- Enable non-expert users to build multi-location, multi-technology energy system models through a guided interface.
- Integrate real-world geographic and power infrastructure data from OpenStreetMap directly into the modelling workflow.
- Cover the entire workflow from model creation to solver execution and results inspection within a single application.
- Produce output that is fully compliant with the Calliope 0.7 framework specification so models can also be run independently on the command line.

## Quality Goals

The following quality goals drive the key architectural decisions of the system, ordered by priority:

1. **Usability** -- A researcher with knowledge of energy systems but limited programming experience must be able to create and run a model without reading source code or YAML documentation.
2. **Correctness** -- The YAML files generated by the exporter must be syntactically and semantically valid Calliope models that the solver can process without modification.
3. **Portability** -- The application must run as a self-contained desktop package on Windows without requiring the user to install separate services or databases beyond a Python environment.
4. **Extensibility** -- New energy technologies, map layers, and data sources shall be integrable with minimal changes to the core application.

## Stakeholders

The main stakeholders relevant for this architecture documentation are:

- **Energy system researchers** at DIT who use the tool to build and experiment with optimization models.
- **Software developers** who maintain and extend the application.
- **Project supervisors** who need an overview of technical decisions and risks.
